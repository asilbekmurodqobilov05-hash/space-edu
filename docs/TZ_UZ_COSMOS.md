# Техническое Задание (ТЗ) — UZ COSMOS

## Обзор Проекта
**UZ COSMOS** — это интерактивная образовательная платформа по астрономии и космонавтике с элементами геймификации и полноценной 3D-симуляцией Солнечной системы в браузере.

## 1. Текущее состояние (Что уже сделано и работает)
### Бэкенд (Django REST Framework)
- Настроена авторизация (`SimpleJWT`) и база данных.
- **Геймификация:** Начисляется опыт (XP), ведется счет топливу (Fuel), отслеживаются ежедневные стрики (Streaks) и бейджи. Управляется через API `/api/v1/gamification/`.
- **Обучение (Courses):** Настроена и заполнена база курсов (8 уровней-миссий от Меркурия до Нептуна). API `/api/v1/courses/` возвращает юниты, уроки и теорию. Учет прогресса идет через `/api/v1/progress/`.
- **Хранилище:** Интегрирован S3/Cloudflare R2 (`backend/base/storage_backends.py`). Картинки загружаются туда (если прописаны ключи в `.env`).

### Фронтенд (React + Vite + Zustand + Three.js)
- **UI/UX:** Единый неоново-фиолетовый (Violet Holographic) дизайн для всех страниц. Настроены переходы, частицы на фоне и стеклянные карточки (Glassmorphism).
- **Обучение:** Раздел Академии (Space Missions) получает данные с сервера. Пользователь проходит теорию и решает квизы; результаты улетают в Django, а опыт начисляется синхронно с бэкендом.
- **3D Симулятор (`/3d-solar-system`):** 
  - Реализована точная Кеплеровская механика для движения планет по реальным орбитам.
  - Текстуры NASA (с картами нормалей, облаками, кольцами) загружаются локально из папки `/public/textures/`. Загрузка мгновенная.
  - Реализован плавный "подлет" камеры (Focus) к планетам.

## 2. Задачи для Команды (Roadmap к Релизу)

### 🧑‍💻 Бэкенд Разработчик (Backend Dev)
1. **Интеграция ИИ (AI Assistant):**
   - Настроить приложение `apps.chat`. Нужно написать логику (View/Service), которая будет принимать вопросы от пользователя с фронтенда, отправлять их в API языковой модели (OpenAI / Gemini / Claude) и возвращать ответ.
   - *Важно:* Передавать контекст (например, если пользователь сейчас проходит урок "Уран", ИИ должен знать об этом).
2. **Скрипты (Parsers) для Новостей и Событий (Опционально):**
   - Написать cron/Celery таски, которые будут автоматически брать "Daily Fact" или космические новости из API NASA (APOD) и сохранять их в таблицу `NewsArticle`.
3. **Безопасность (Security):**
   - Добавить валидацию (Throttle/Rate Limits) на эндпоинты авторизации и ИИ-чата.

### ⚛️ Фронтенд Разработчик (Frontend Dev)
1. **Мультиязычность (i18n):**
   - Сейчас в базе Django есть поля `title_en`, `title_uz`, `title_ru`. На фронтенде нужно доработать логику так, чтобы при переключении языка в `useTranslation` (или в `i18next`) данные с бэкенда отображались на выбранном языке.
2. **Связь AI с Бэкендом:**
   - Компонент `AskCosmos.jsx` сейчас работает как визуальная заглушка. Нужно привязать его к эндпоинту ИИ-чата (созданного бэкендером).
3. **Мобильная адаптивность в 3D:**
   - Симулятор (`SolarSystemView.jsx`) отлично работает мышкой, но на смартфонах `OrbitControls` могут конфликтовать со свайпами. Необходимо протестировать управление пальцами и добавить экранные кнопки (Zoom + / Zoom -, Focus).

### 📝 Контент-менеджер / Переводчик (Content & Translations)
1. **Заполнение Базы Данных (Django Admin):**
   - Зайти в админку (раздел "Courses") и перевести тексты уроков (в поля `content_uz`, `content_ru`).
   - Добавить реальные "Космические новости" (`apps.news`) и события (`apps.events`). Прикрепить красивые обложки (они сами загрузятся в Cloudflare).
2. **Магазин и Беджи (Market & Badges):**
   - Придумать скины для космонавта или бонусы, которые можно будет купить за "Топливо" (Fuel) в `apps.market`.
   - Заполнить карточки профессий (Careers) реальным текстом об узбекских и мировых инженерах.

### ⚙️ DevOps / Системный Администратор (Infra)
1. **Настройка Cloudflare R2:**
   - Создать Bucket в Cloudflare.
   - Записать ключи (Access Key, Secret Key, Endpoint) в `.env` файл на рабочем (Production) сервере.
2. **Домен и SSL:**
   - Настроить Nginx/Gunicorn для раздачи бэкенда.
   - Подключить фронтенд (React build) и бэкенд на рабочий домен (например, `api.uzcosmos.uz` и `uzcosmos.uz`).
3. **База данных:**
   - Перевести проект с `db.sqlite3` на полноценный **PostgreSQL**.

## 3. Архитектурная памятка
* **Хранение медиа:** Любые изображения, загруженные через админку, автоматически отправляются в Cloudflare R2 (настраивается в `settings/base.py`). В базе сохраняется лишь URL.
* **Текстуры 3D (R3F):** Все карты планет и космоса лежат **статично** в репозитории `frontend/public/textures/` для обеспечения высокой скорости работы WebGL. Их не нужно загружать в Cloudflare.
* **Управление состоянием:** На фронтенде всё глобальное состояние (Авторизация, Геймификация, ИИ-контекст) хранится в хранилищах Zustand (`useAuthStore`, `useGamificationStore`, `useLearningStore`). При логине `fetchMe` автоматически синхронизирует локальное Zustand хранилище с базой Django.
