**Комиссия:** Государственная проверка — код и контент будут изучены строго. Ничего лишнего, всё должно работать, данные — реальные.

### 2.2 Asilbek — Игра и визуальные ассеты

#### Задача A: Разработка и улучшение игры SpaceRun

**Текущее состояние:** Игра работает в браузере, базовая логика на Zustand.

**Требования к улучшению:**

**Физика:**
- Гравитация: ракета ускоряется вниз с постоянным `g = 0.3` (настраиваемый)
- Тяга: при нажатии — ускорение вверх с силой `thrust = 0.8`
- Трение воздуха: лёгкое торможение горизонтали (`drag = 0.98`)
- Коллизии: точные (AABB или circle-based, не пиксельные)

**Производительность:**
- Игра должна работать стабильно при 60 FPS на среднем железе
- Использовать `requestAnimationFrame` правильно (delta time)
- Не создавать новые объекты в game loop — переиспользовать пул объектов
- Canvas или CSS transforms — на выбор разработчика, но без WebGL (Three.js — только для 3D секций)

**Безопасность и надёжность:**
- Нет утечек памяти (clearInterval/cancelAnimationFrame при unmount)
- Обработка паузы/возобновления (вкладка теряет фокус)
- Обработка изменения размера окна (resize)
- Защита от читов: счёт хранится только на сервере (отправляется через API после сессии)

**Геймплей:**
- Препятствия: астероиды, случайно генерируемые
- Бонусы: монеты (топливо +10), звёзды (XP +5)
- Сложность: увеличивается каждые 30 секунд (скорость спауна, скорость движения)
- Game Over: экран с результатом, кнопка "Снова" и "В меню"
- Рекорд: сохраняется в Zustand + отправляется в API

**Интерфейс игры:**
- HUD: текущий счёт, монеты, уровень сложности
- Пауза: кнопка или Escape
- Мобильная поддержка: тап по экрану = тяга

**Файлы для изменения:**
```
frontend/src/game/
├── SpaceRun.jsx       ← основная логика
├── useGameStore.js    ← Zustand store игры
└── components/        ← если нужно разбить на компоненты
```

**LoginView.jsx — требования:**
- Поля: Email, Пароль
- Валидация: email формат, пароль мин. 6 символов
- Submit → `POST /api/v1/auth/login/`
- При успехе: сохранить `accessToken` + `refreshToken` в Zustand + localStorage
- При ошибке: показать сообщение "Неверный email или пароль"
- Ссылка "Нет аккаунта? Зарегистрироваться" → `/register`
- Дизайн: космическая тема, тёмный фон, анимация Framer Motion при входе

**RegisterView.jsx — требования:**
- Поля: Имя пользователя, Email, Пароль, Подтверждение пароля
- Валидация: все поля обязательны, пароли должны совпадать, email уникален
- Submit → `POST /api/v1/auth/register/`
- При успехе: автоматический логин (сохранить токены), редирект на `/learn`
- При ошибке: показать конкретную ошибку с бэкенда
- Ссылка "Уже есть аккаунт?" → `/login`

**useAuthStore.js — требования:**
```js
// Состояние:
{ user, accessToken, refreshToken, isAuthenticated }

// Действия:
login(tokens, userData)   // сохранить токены + user
logout()                   // очистить всё
updateUser(data)           // обновить профиль
```

**api.js — требования:**
```js
// Axios instance:
// - baseURL из import.meta.env.VITE_API_URL
// - Interceptor запроса: добавить Authorization: Bearer <token>
// - Interceptor ответа: при 401 — автообновление через /auth/token/refresh/
//   если refresh тоже упал — logout() + редирект на /login
```

**ProtectedRoute.jsx — требования:**
```jsx
// Если isAuthenticated === false → <Navigate to="/login" />
// Иначе → <Outlet />
```

**ProfileView.jsx — требования:**
- Показывает: аватар, имя, email, уровень, XP, топливо, streak
- XP-бар с прогрессом до следующего уровня
- Список полученных значков (badges)
- Радарная диаграмма навыков (SVG, без внешних библиотек)
- Кнопка "Редактировать" → форма изменения имени и аватара
- Данные загружаются из `GET /api/v1/auth/me/` + `GET /api/v1/gamification/profile/`

## 3. Технические требования ко всем задачам

### 3.1 Код

- Все файлы: `.jsx` / `.js` (не TypeScript)
- Импорты: только через `@/` алиас (не `../../`)
- Один default export на файл, имя = имени файла
- Нет `console.log` в итоговом коде
- Нет мёртвого кода, нет закомментированных блоков
- Файл max ~250 строк — если больше, разбить

### 3.2 Стиль

- Tailwind CSS классы (не inline styles)
- Цветовая схема: тёмная (space theme), акценты — синий/фиолетовый/золотой
- Анимации: Framer Motion (`motion.div`, `AnimatePresence`)
- Иконки: lucide-react

### 3.3 Многоязычность

При добавлении ЛЮБОГО текстового контента:
1. Добавить ключ в `src/locales/en.json`
2. Добавить ключ в `src/locales/uz.json`
3. Добавить ключ в `src/locales/ru.json`
4. В компоненте использовать `const { t } = useTranslation()`

### 3.4 API интеграция

- Все запросы через `src/lib/api.js` (axios instance)
- Нет прямых `fetch()` вызовов в компонентах
- Состояния загрузки: показывать спиннер или скелетон
- Ошибки API: показывать понятное сообщение пользователю

### 3.5 Мобильность

- Все новые компоненты должны быть адаптивны
- Тестировать при ширине 375px (iPhone SE)
- Touch события для мобильной игры
