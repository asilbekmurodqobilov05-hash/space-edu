# 🔍 Детальный Технический Анализ Проекта: UZ COSMOS

Этот документ содержит глубокий разбор реализации ключевых функций платформы, их логику и текущее состояние интеграции.

---

## 1. 🔐 Система Авторизации (Auth System)
*   **Бэкенд (`apps/accounts`):**
    *   **Технологии:** Django REST Framework + SimpleJWT.
    *   **Логика:** Кастомная модель `User`. При регистрации `username` автоматически создается из префикса email. Пароли проверяются встроенными валидаторами Django (длина, сложность).
    *   **API:** Готовы эндпоинты `/register/`, `/login/`, `/logout/`, `/me/` (профиль) и `/delete/`.
*   **Фронтенд (`views/auth`):**
    *   **UI:** Полностью реализованы `LoginView` и `RegisterView`. Дизайн на базе `GlassCard` с эффектами свечения.
    *   **Интеграция:** Подключены к API через `api.js`. Используют `useAuthStore` для хранения токенов.

## 2. 🌌 Симулятор Солнечной Системы (Solar Simulator)
*   **Логика:** Реализована в `SolarSystemView.jsx` с использованием `react-three-fiber`.
*   **Физика:** Движение планет рассчитывается по Кеплеровским параметрам (эксцентриситет, наклонение и т.д.) из файла `solarSystemData.js`.
*   **Текстуры:** Используются локальные текстуры высокого разрешения из `/public/textures/` (NASA Texture Maps).
*   **Связь:** Полностью клиентская (статичная), что оправдано для физически точной модели.

## 3. 📚 Академия и Контент (Academy)
*   **Структура БД:** Иерархия `Level (Уровень) -> Unit (Блок) -> Lesson (Урок)`.
*   **Мультиязычность:** Каждая модель имеет поля `_en`, `_uz`, `_ru`. 
*   **API Интеграция:** `LearnView.jsx` загружает список уровней через `GET /api/v1/courses/levels/`.
*   **Точка роста:** Нужно настроить фронтенд на динамический выбор поля (например, `title_ru` вместо `title_en`) в зависимости от выбранного языка в приложении.

## 4. 🏆 Геймификация (Gamification)
*   **XP & Levels:** Синхронизированная формула: `Level = floor(sqrt(xp / 100)) + 1`. 
*   **Fuel (Топливо):** Реализовано как ресурс для доступа к миссиям. Лимит — 1000 единиц.
*   **Бейджи (Badges):**
    *   **Бэкенд:** Сервис `check_and_award_badges` выдает награды за XP, стрики и завершенные уроки.
    *   **Фронтенд:** Store `useGamificationStore` дублирует часть логики для мгновенного фидбека в UI.
*   **Синхронизация:** При входе выполняется `syncFromAPI`, обновляющий локальное состояние из БД.

## 5. 🎮 Космическая Игра (SpaceRun)
*   **Стек:** Three.js + Zustand.
*   **Функционал:** Бесконечный раннер с астероидами и сбором энергии. 
*   **Интеграция:** Сейчас очки (`score`) хранятся только в `spaceArcadeStore.js`.
*   **Задача:** Подключить сохранение рекордов в профиль пользователя на бэкенде.

## 6. 🛠 Техническая Архитектура (The Glue)
*   **API Клиент (`lib/api.js`):** Настроен Axios с интерцепторами. Если `access`-токен протухает, клиент автоматически идет за новым через `refresh`-токен, не прерывая работу пользователя.
*   **Global Stores:** Zustand используется для управления всем состоянием. Middleware `persist` сохраняет данные в `localStorage`.

1. 🔐 Система Аутентификации (Register & Login)
Backend: Реализован на Django REST + SimpleJWT. Интересная деталь: username генерируется автоматически из email (например, user@mail.com → user), что упрощает UX. Есть валидация возраста (дата рождения должна быть в прошлом) и совпадения паролей.
Frontend: Файлы LoginView.jsx и RegisterView.jsx уже существуют и полностью готовы. Они используют красивые анимации framer-motion и компоненты GlassCard.
Связь API: Фронтенд уже отправляет данные на /api/v1/auth/login/ и /api/v1/auth/register/. После входа токены сохраняются в useAuthStore, а состояние синхронизируется с бэкендом.
2. 🌌 Симулятор Солнечной Системы (Simulator)
Логика: Находится в SolarSystemView.jsx. Использует реальные орбитальные параметры J2000 (эксцентриситет, наклонение, большая полуось).
Данные: Все параметры планет вынесены в solarSystemData.js.
Визуал: Текстуры подгружаются из /public/textures/. Реализован механизм выбора планеты и "подлета" камеры к ней.
Связь API: На данный момент симулятор работает полностью на клиентской стороне (статичные данные), так как параметры планет не меняются.
3. 🎮 Игра "SpaceRun"
Логика: Это полноценная Three.js игра в жанре бесконечного раннера (SpaceRunScene.jsx).
Управление: Реализовано через клавиатуру (AD / Стрелки). Есть система препятствий (астероидов) и сбора энергии.
Score/XP: Очки в игре считаются в spaceArcadeStore.js.
Связь API: Пока отсутствует. Очки за игру сохраняются только локально. Нужно добавить эндпоинт на бэкенде для сохранения рекордов.
4. 📚 Система Обучения (Academy)
Backend: Мощная структура Level -> Unit -> Lesson -> LessonSection. Все тексты поддерживают три языка (title_ru, title_uz, title_en).
Frontend: LearnView.jsx уже подключен к API и загружает уровни с сервера.
Проблема: В коде LearnView сейчас жестко прописано отображение английских полей (level.title_en). Нужно заменить на динамический выбор в зависимости от текущего языка пользователя.
5. 🏆 Геймификация (XP, Level, Badges)
Синхронизация: Это самая сильная часть. Формула уровня floor(sqrt(xp/100)) + 1 идентична и на Django, и на React. Это гарантирует, что пользователь увидит один и тот же уровень везде.
Топливо (Fuel): Лимит в 1000 единиц соблюдается на обеих сторонах.
Бейджи:
Backend: Есть сервис check_and_award_badges, который проверяет условия (XP, стрики, количество уроков) при завершении урока.
Frontend: В useGamificationStore есть дублирующая логика checkBadges.
Совет: Лучше оставить выдачу бейджей только на бэкенде, а фронтенд должен просто получать список новых бейджей в ответе от API.
6. 🔗 Технический "Клей" (API & Stores)
Axios Client: В lib/api.js настроен продвинутый клиент. Он умеет автоматически обновлять access-токен через refresh, если сессия истекла. Это уровень серьезного продакшн-приложения.
Zustand Stores: Состояние разделено на useAuthStore (токены), useGamificationStore (прогресс) и useUserStore (настройки). Все они используют persist, то есть данные не пропадают при перезагрузке страницы.
